1 Abstract

This proposal revises COMP6772 as proposed in P6772R1.

2 Motivation

When bikeshedding a good name for COMP6772 with Hayden and John, we were initially under the impression that they might share a common name, with COMP6772 prefixing โAdvancedโ. I suggested the following pairs of names:

  • โMulti-Paradigm Programmingโ/โAdvanced Multi-Paradigm Programmingโ

Hayden pointed out that COMP6771 is a basic course that focusses on students getting their heads specifically around C++. That is, COMP6771 is intrinsically a C++ course, which students take to learn C++, and that should be reflected in its name. He also noted that if we drop โAdvancedโ from COMP6771โs name and adopt it for COMP6772, students who took COMP6771 Advanced C++ Programming might get confused if a second Advanced C++ Programming course popped up (especially if itโs got different content!). As such, it would be a good idea for CSE to zombify Advanced C++ Programming, if COMP6771 gets renamed.

An astute reader should notice that all the pairs above have now been struck: the latter three donโt have โC++โ in their name, and the first one has an invalid name for COMP6772. Hayden also made the observation that the relationship between COMP6771 and COMP6772 is kinda like COMP1917 and COMP1927 (sic): 1917 is about โlearning the languageโ, and 1927 gives a little bit more about the language, but is mainly focussed on using it, and its application. I strongly agree.

With Haydenโs advice and observations in mind, P6772R1โs proposed material and my background in generic programming, the best name for the course is Generic Programming. P6772R1 doesnโt propose a course on generic programming, which is a bit of a problem. Fortunately, it can be slightly adjusted to be a course on this topic, without damaging the integrity of the course thatโs already been proposed.

By removing the content on allocators, which are have less utility anyway, and narrowing the amount of time spent on software engineering practices from two weeks to one week, weโre able to have a course that spends the majority of its time exploring how to leverage generic programming either by using abstractions or synthesisng them.

3 Changes to the course design

3.1 Tooling

Tooling has been reduced from two weeks of lectures (1, 4) to just one (4).

3.2 Program composition

Program composition has been expanded from two weeks of teaching to three. Weโll also builds upon the COMP6771 STL week to help students develop algorithm intution.

3.3 Containers design

This section of the course has been completely removed. This is both the largest change and the most impactful one.

3.4 Generic programming

This topic has been expanded from two weeks to four. The second assessment now focusses on generic programming instead of container design. Since the course is now called โGeneric Programmingโ, this module will either be renamed or split into two parts, but thatโs not really something that needs to happen right now.

4 Proposed changes

This course continues teaching C++ where COMP6771 leaves it. Deepens knowledge learnt in COMP6771; introduces advanced topics that will reshape the way in which one writes C++ programs.

Generic programming is the study of algorithms and data structures, where we make them available to as many contexts as possible, without losing efficiency.

This course teaches the fundamentals of generic programming, both for use and implementation, using C++. In a way, this course picks up C++ where COMP6771 leaves it. Introduces program composition with ranges and coroutines; deepens algorithm intuition. Studies how to generalise algorithms and describe requirements.

Detailed study of complex C++ tools for better, faster, and correct programs. Program composition in C++. Building containers that isare performance-sensitive. Generic programing.

4.1 Objectives

4.1.1 Knowledge and understanding

Students will develop knowledge and understanding of:

  • contemporary C++ software engineering practices
  • the philosophy of C++ generic programming
  • algorithms and abstraction
  • social and ethical issues, and their effect on software engineering

4.1.2 Skills

Students will develop skills in:

  • using C++ tools
  • more in-depth C++ programming techniques
  • generating generic algorithms

Outcomes
